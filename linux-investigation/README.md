# 🐧 Linux System Investigation

## 🎯 Objective
Investigate Linux systems to identify sensitive files, analyze system behavior, and perform basic security and operational analysis using command-line tools.

---

## 🛠 Tools Used
- Linux Command Line
- File & Permission Analysis
- Log Investigation
- System Enumeration

---

## 🔍 Activities Performed

### 📂 File & Directory Investigation
- Identified sensitive files and directories using Linux commands.
- Investigated file permissions and ownership settings.
- Analyzed system directories, hidden files, and configuration files.

### 🖥 System Enumeration
- Performed system and service enumeration to identify running processes and active services.
- Investigated scheduled tasks, cron jobs, and startup configurations.
- Reviewed system information, user accounts, and environment variables.

### 📄 Log & Security Investigation
- Analyzed authentication and system logs for suspicious activity.
- Investigated failed login attempts and abnormal system behavior.
- Reviewed command-line history and service-related logs.

### ⚙️ Command-Line Operations
- Used Linux commands for file searching, filtering, and analysis.
- Performed basic troubleshooting and system investigation tasks.
- Investigated permissions, access control, and service configurations.

---

## 🔎 Example Commands Used

```bash
find / -name "*.txt"
grep "failed" auth.log
ls -la
cat file.txt
pwd
ps aux
crontab -l
```

---

## 🧭 Investigation Methodology
1. Enumerated files, users, and services.
2. Investigated permissions and sensitive directories.
3. Reviewed logs and authentication activity.
4. Identified suspicious or misconfigured system components.
5. Documented findings and analyzed system behavior.

---

## 🚩 Key Findings
- Identified sensitive files and exposed system information.
- Observed insecure permissions and configuration weaknesses.
- Investigated suspicious authentication and service activity.

---

## 🛡 Security Recommendations
- Apply least privilege access controls.
- Restrict unnecessary services and permissions.
- Monitor authentication logs and scheduled tasks.
- Regularly review sensitive files and system configurations.
